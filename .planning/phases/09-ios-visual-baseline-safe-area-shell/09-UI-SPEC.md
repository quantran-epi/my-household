# Phase 9 — UI Design Contract

> Visual and interaction contract for the iOS Visual Baseline & Safe-Area Shell phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase nature:** This is a foundational token + shell phase, NOT a feature-screen phase. The "screens" are the shared sheet/picker layer (FastOverlay, SheetPicker) and the app shell (bottom nav, cooking pill, header, content area). This contract codifies the EXISTING Phase 7/8 aesthetic into named tokens (CONTEXT D-02/D-03) — it does NOT introduce a new visual language. Every value below is sourced from a shipped literal in the codebase; the deliverable is centralizing them, not redesigning them.

## Token Module Contract (IOS-01 — primary deliverable)

The module is the single source of truth (D-02). Existing duplicated literals across `FastOverlay.tsx`, `BottomTabNavigator.tsx`, `CookingPill.tsx`, `Content.tsx`, `SheetTrigger.tsx`, `App.tsx` are promoted into the named tokens below, then those call sites are refactored to read from the module. Categories are deliberately lightweight (D-03, YAGNI) — no color-system overhaul beyond promoting the existing purple set.

Core scalar tokens that AntD owns (primary color set, base font, radius) are ALSO fed into `ConfigProvider theme.token` in `App.tsx` from this same module (D-04), so AntD-rendered surfaces (incl. SheetDatePicker's hosted calendar panel) and hand-built sheet chrome align from one definition. Tokens AntD has no slot for (sheet gradient, safe-area, touch-target) live only in the TS module.

## Spacing Scale

Declared canonical scale (multiples of 4, anchored to the iOS HIG ~8pt rhythm):

| Token | Value | Usage (from existing literals) |
|-------|-------|--------------------------------|
| xs | 4px | Icon/label gaps (nav `gap:2-4`), grabber top inset |
| sm | 8px | Compact element spacing, action-row gap, title gap, safe-area base inset |
| md | 12px | Content padding (`CONTENT_PADDING=12`), sheet body row gaps, header gap |
| lg | 16px | Default body/section padding (sheet body `padding:16`, row `11px 16px`) |
| xl | 24px | Section breaks |
| 2xl | 32px | Empty-state vertical padding (SheetSelect no-results `32px 16px`) |
| 3xl | 48px | Page-level spacing |

Exceptions (existing idioms preserved, not re-derived — promote-what-exists):
- `11px` horizontal padding on the picker trigger (`SheetTrigger.tsx:31`) — intentional AntD-input parity; keep as the trigger's own padding token, do not round to 12.
- `10px` paddings in nav/action rows and `6px`/`5px` decorative insets (grabber pill height 5px, dock padding `6px 8px`) — decorative chrome, kept verbatim.
- `1px` hairline borders are a border token, not a spacing step (see Surface tokens).

## Typography

Base body is 18px via AntD `fontSize:18` (already shipped). Declared roles (4 sizes, 2 weights):

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Caption | 12px | 400 regular | 1.4 (~16px) | Secondary/meta text, progress sub-labels |
| Control | 16px | 400 regular | 22px (~1.375) | Picker trigger text, sheet selectable rows (`SheetTrigger`, `SheetSelect` rows) |
| Body | 18px | 400 regular | 1.5 | App-wide default body (AntD base) |
| Title | 18px | 650 semibold | 1.3 | Sheet/modal/drawer titles (`shellTitleStyle` weight 650, color `#2f2545`) |

Weights: regular **400** + semibold **650** (existing `shellTitleStyle`/active-nav weight).

Exceptions (decorative, out of the 2-weight rule by intent):
- Inactive nav label uses **500** medium (`BottomTabNavigator` `_labelStyles`); active uses 650. Acceptable — it is the same nav family expressing state.
- Cooking pill uses **700/750** and 10–14px sizes (`CookingPill.tsx`) — a deliberately bold decorative floating element; not part of the standard type ramp and not re-tokenized for weight here.
- Nav labels render at 10px (`fontSize:10`) — sub-caption chrome size, kept verbatim as a nav-specific constant.

## Safe-Area Shell Contract (IOS-02)

`viewport-fit=cover` is ALREADY shipped (`public/index.html:7`) — consumed here, NOT re-added. Establish ONE safe-area convention and route all sticky chrome through it (D-07). The reference implementation is `BottomTabNavigator.tsx:37` (`calc(8px + env(safe-area-inset-bottom))`); all sticky bottom chrome converges on this `calc(... + env())` shape.

| Token / helper | Contract |
|----------------|----------|
| `safeAreaInset.bottom(base)` | Returns `calc(${base}px + env(safe-area-inset-bottom))`. Function vs string-set is Claude's discretion (CONTEXT). |
| `safeAreaInset.top(base)` | `calc(${base}px + env(safe-area-inset-top))` for sticky top chrome |
| `layout.headerHeight` | 76 (promote `HEADER_HEIGHT`) |
| `layout.bottomNavHeight` | 80 (promote `BOTTOM_NAV_HEIGHT`; nav container min-height is 88) |

Required fixes (the core IOS-02 work):
- **`Content.tsx` (D-05):** replace `height: calc(100vh - 156px)` / `maxHeight: calc(100dvh - 156px)` mismatch with token-driven height that subtracts `env(safe-area-inset-top/bottom)` and uses **`dvh` as primary unit, `vh` fallback only**. The two height rules must agree.
- **`CookingPill.tsx` (D-06):** `position:fixed; bottom:76` has NO safe-area inset — rebase `bottom` on bottom-nav height **plus** `env(safe-area-inset-bottom)` from tokens so it clears the home indicator and never collides with the nav.
- Left/right insets applied only where horizontally-pinned elements exist (landscape); bottom is the IOS-02 priority.
- `App.css` `min-height:100vh` and `ErrorBoundary` `100vh` → `dvh`+inset is Claude's discretion (low-risk consistency win; do if cheap).
