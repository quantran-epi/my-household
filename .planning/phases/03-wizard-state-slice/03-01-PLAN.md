<objective>
Create the wizard state layer: a `WizardState` model and a `WizardReducer` RTK slice with per-step commit semantics, plus co-located reducer unit tests. The slice holds the current step (string key), a per-step answer model, and a session status, with action creators to commit answers, advance/go-back, resume, restart, and complete.

Purpose: This is the persisted state foundation for the meal-planning wizard (FND-03, WIZ-06). Per-step commits are the durability point ŌĆö each commit lands in state that redux-persist serializes to the already-persisted `personal` root, so a forced reload mid-flow preserves prior answers. No UI is built here; Phase 4 consumes this slice.

Output: `src/Store/Models/Wizard.ts`, `src/Store/Reducers/WizardReducer.ts`, `src/Store/Reducers/WizardReducer.test.ts`.
</objective>
