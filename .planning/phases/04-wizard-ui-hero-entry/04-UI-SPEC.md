# Phase 4 — UI Design Contract

> Visual and interaction contract for the guided meal-planning wizard + Home hero reframe. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack note:** This is a brownfield Ant Design v5 app. Tech-stack replacement is explicitly out of scope (REQUIREMENTS.md "Out of Scope"). The shadcn gate is **not applicable** — all tokens below are extracted from the existing `ConfigProvider` theme (`src/App.tsx`) and live component code (`DashboardHero` in `Dashboard.screen.tsx`). New wizard screens MUST match the surrounding inline-style + antd-token idiom, not introduce a new system.
>

App-wide `AppCopy` migration is Phase 5 (COPY-03..05) — write new wizard copy in Vietnamese **inline** this phase. This contract specifies the desktop-and-acceptable-on-mobile baseline only.

## Spacing Scale

Declared values for new wizard surfaces (multiples of 4). The existing codebase uses tighter ad-hoc values (6/10/14); for new wizard screens normalize to this 4-based scale:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, chip inner padding |
| sm | 8px | Compact element spacing, gap between option rows |
| md | 16px | Default padding inside step cards, gap between step title and body |
| lg | 24px | Step screen outer padding, gap between question and answer controls |
| xl | 32px | Major vertical breaks (progress → question → CTA zones) |
| 2xl | 48px | Result-step section break (suggestion list → add-to-day affordance) |

Exceptions:
- Hero card internal padding stays at the existing **14px** to match `DashboardHero` unchanged sections (D-09 reframes the hero in place, not a full restyle).
- ~44px touch-target sizing is **deferred to Phase 5 (MOB-02)** — do not retrofit it here.

## Typography

Exactly 4 roles, 2 weights (400 regular, 600 semibold). New wizard screens use these; do not reach for the heavier 650/700/750 weights the legacy dashboard uses.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display (step question / hero CTA label) | 28px | 600 | 1.2 |
| Heading (step title, result dish name) | 20px | 600 | 1.25 |
| Body (option labels, descriptions, dish meta) | 16px | 400 | 1.5 |
| Label (progress text, captions, skip hint) | 13px | 600 | 1.4 |

## Color

60/30/10 split for new wizard surfaces, anchored to the existing primary `#7436dc`:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff | Step screen background, card surfaces, content panels |
| Secondary (30%) | #f5f5f5 | Step cards / option rows / picker panels, unselected chips, nav surfaces |
| Accent (10%) | #7436dc | See reserved list below only |
| Destructive | #ff4d4f (antd `colorError`) | "Bắt đầu lại" wizard reset confirmation only |

Accent (#7436dc) reserved for:
- The Home hero "Hôm nay ăn gì?" primary CTA (white-on-gradient per existing `DashboardHero` treatment)
- The wizard step primary action button ("Tiếp tục" / "Xem gợi ý")
- The result "Thêm vào hôm nay" primary add button
- The progress-indicator fill / current-step marker
- Selected state on ingredient chips and option rows (selected border + tint)

Accent is NOT used for: back button, "Tùy bạn" skip, secondary/ghost actions, "Chọn ngày khác" — these are neutral/ghost styling.

## Interaction Contract (phase-specific, for executor)

Load-bearing interaction rules the executor must honor — derived from CONTEXT.md (D-01..D-10) and 04-RESEARCH.md.

- **One question per screen (WIZ-02):** the wizard container reads `selectWizardStep` and renders exactly one step (`ingredients` → `preferences` → `result`). Progress indicator + back affordance are persistent chrome above the step body. Back dispatches `goBackWizardStep(prevKey)`; advance commits via `commitWizardAnswer` then `advanceWizardStep(nextKey)`.
- **Skip never dead-ends (WIZ-03 / D-04):** "Tùy bạn" commits a sensible default (ingredients → `[]` → full-catalog fallback; preference → neutral) then advances. Every step shows the skip affordance as a low-emphasis (neutral, not accent) action.
- **Result always actionable (WIZ-04 / D-08):** fallback ladder — matches → top few (recommend N=5); no match but catalog non-empty → full-catalog suggestion list with the neutral no-match note; empty catalog → empty state routing to "Thêm món đầu tiên". `DishScorer` output is already sorted — do not re-sort; presentation only.
- **Add-to-meal single-tap (WIZ-05 / D-07):** "Thêm vào hôm nay" is the dominant action, defaults to today + `dinner` slot, fires `message.success` then `completeWizard()`. "Chọn ngày khác" reveals a `DatePicker` inside a `@components/Sheet` — secondary, neutral styling.
- **In-step pickers use bottom Sheet (D-01):** ingredient picker, preference tag picker, and day picker host inside `@components/Sheet` (respect `zIndexPopupBase: 4000`). Reuse `IngredientPickerWidget` as-is.
- **Reachability preserved (NAV-02/04):** bottom-nav center "Nấu gì?" repoints to the wizard route; the in-place suggester stays reachable via its `/dish-suggester` sidebar entry — do not orphan it. Preserve `data-testid="bottom-tab-suggester"`.
- **Hero reframe in place (D-09):** add the dedicated "Hôm nay ăn gì?" top CTA to `DashboardHero`; keep existing dashboard sections/metrics intact (entry-points-only, D-10).
