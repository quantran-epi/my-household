| Boundary | Description |
|----------|-------------|
| Persisted wizard slice → UI | The container renders whichever step the rehydrated `currentStep` reports; a tampered/unknown step key could mis-route the UI |
| URL route → app shell | `/meal-planning/wizard` is an unauthenticated client route (no auth in this app by design) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering | Persisted `currentStep` key | mitigate | `selectWizardStep` falls back to `WIZARD_FIRST_STEP` (Phase 3 defensive selector); `idx = Math.max(0, WIZARD_STEPS.indexOf(step))` clamps unknown keys to the first step rather than rendering nothing |
| T-04-07 | Denial of Service | Step render loop | mitigate | Exactly one step renders per key via discrete equality checks; no recursion or unbounded mapping |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new package installs — react-redux, react-router-dom, antd, existing `@components`/`@store` already present |
</threat_model>
