<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input -> PIN gate | PIN entry/validation hosted in the swapped confirmation surface |
| user action -> data backup | Backup/restore triggers read/write of persisted local data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Information Disclosure | PIN value rendered/handled in the swapped Sheet | mitigate | Move the EXISTING PIN input + validation logic verbatim into the Sheet body — do not log, echo, or persist the PIN; no new storage. Sheet provides the same masked input as the Modal did. |
| T-05-06 | Tampering | Backup/restore confirm moved from Modal `onOk` to Sheet Button | mitigate | Preserve the exact original restore handler behind an explicit primary-CTA tap; no implicit confirm; destructive restore keeps a `danger` affordance. |
| T-05-07 | Repudiation | Copy key in glossary references a secret | accept | Glossary review-only; this plan adds no credential-bearing copy key (Glossary.ts security note). |

No package installs, no network calls. The PIN/backup logic is moved, not re-implemented — behavior preserved.
</threat_model>
