# Phase 8 — UI Design Contract

> Visual and interaction contract for the reusable `@components/SheetPicker` layer (SheetSelect, SheetMultiSelect, SheetDatePicker, SheetActionMenu). Built on the Phase 7 native `<Sheet>`; tested in isolation; no real call sites migrated this phase (Phases 10-11). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This contract locks the visual/interaction details CONTEXT.md left to "Claude's Discretion" while honoring all locked decisions D-01..D-12. It does NOT re-spec the Phase-7 sheet chrome (grabber, drag-to-dismiss, safe-area, z-index stacking) — that is owned by Phase 7. It matches the Phase-7 aesthetic rather than inventing new tokens; the cohesive iOS token system lands in Phase 9 (IOS-01..03).

## Design System

| Property | Value |
|----------|-------|
| Tool | none (CRA + craco; AntD 5.16.1 is the component library — no shadcn, not React/Vite scaffold target) |
| Preset | not applicable |
| Component library | AntD 5.16.1 (`Checkbox`, `Input`, `DatePicker`/`RangePicker` panel, `Button`, `@ant-design/icons`) + in-repo `@components/Sheet` (Phase 7) |
| Icon library | `@ant-design/icons` (already a dependency) — `CheckOutlined` (single-select check), `CloseCircleFilled` (trigger clear ×), `SearchOutlined` (search field), `CalendarOutlined`/`DownOutlined` (trigger affordances) |
| Font | System stack via `--app-font-family`: `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif` (defined in `src/index.css`) — inherit, do not restate per-component |

**shadcn gate:** Not applicable. This is a CRA project using AntD; no `components.json`, no `tailwind.config`. Registry safety gate is not applicable (no third-party registries).

**Styling approach (matches Phase 7):** inline `style` objects + a single injected `<style>` block per component for what the `style` prop can't express (pseudo-states, scoped popup overrides). No CSS modules. Render inside the portal'd `<Sheet>`. Never pass an explicit `zIndex` to `<Sheet>` (breaks Phase 7 token stacking).

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap inside a row, count-badge inset |
| sm | 8px | Search field margin, gap between checkbox and label, ActionMenu card-to-Cancel gap base unit |
| md | 16px | Sheet body padding (inherited from `Sheet` body `padding:16`), row horizontal padding, section spacing |
| lg | 24px | Group separation, ActionMenu card-to-detached-"Hủy" gap |
| xl | 32px | Reserved — large vertical breaks if a sheet body needs them |

Exceptions (touch-target, not spacing-rhythm — intentional per IOS-03 bar built early):

- **Row min-height: 44px** — every tappable option row (SheetSelect, SheetMultiSelect), every ActionMenu action row, the "Bỏ chọn"/"Hôm nay" quick-action rows, and the closed trigger all meet the ≥44px thumb-zone bar now, even though the formal iOS token pass is Phase 9. Vertical padding inside a 44px row is 11px when paired with 22px line-height; rows grow taller (wrap) for long labels but never shorter than 44px.
- **Trigger min-height: 44px** — overrides AntD's default `controlHeight` (32px) so the closed control is thumb-friendly. Border/radius/text still match the AntD input box (see Trigger spec) so converted screens read as the same control.

Larger 8-point steps (48/64) are unused this phase (no page-level layout here — these are in-sheet components).

## Typography

System font stack (above). Two weights only: regular 400, semibold 600.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Row label / option text | 16px | 400 | 1.375 (22px) |
| Trigger value / placeholder | 16px | 400 (placeholder uses muted color, not a different weight) | 1.375 (22px) |
| Section header / "Đã chọn (N)" / search caption / sheet title | 14px | 600 | 1.43 (20px) |
| Action button label ("Xong (N)", "Hủy", "Hôm nay") | 16px | 600 | 1.5 (24px) |

Notes:
- 16px row/value text is the iOS-friendly minimum that avoids the iOS Safari focus-zoom on inputs (the search `<Input>` must be ≥16px to not trigger zoom).
- Heading/label line-height 1.2-1.45; body/row 1.375; button 1.5. Within the 1.2-1.5 band the checker expects.
- Destructive ActionMenu rows keep weight 400 (color, not weight, signals danger — see Color).

## Color

Anchored to the Phase-7 sheet aesthetic. The sheet surface gradient is owned by Phase 7 and reused verbatim (not re-tokenized here).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Sheet surface `linear-gradient(180deg, #f5f0ff 0%, #ffffff 42%)` (effective base `#ffffff`) | Sheet body background (inherited from `Sheet`), row backgrounds, trigger fill |
| Secondary (30%) | `#f8fafc` (slate-50, already used in `Form/Select.tsx` selected-options panel) | "Đã chọn (N)" summary strip, search-field container, ActionMenu grouped card surface, pressed/hover row state `rgba(116,54,220,0.06)` |
| Accent (10%) | `#7436dc` (Phase-7 primary purple) | Selected-row checkmark + selected-row label, checked checkbox fill, "Xong (N)" primary button fill, "Hôm nay" quick-action emphasis, focused search border, trigger focus ring, today-cell highlight in the embedded calendar |
| Destructive | `#ff4d4f` (AntD `colorError`) | ActionMenu `danger` row label/icon only |

Accent reserved for: the single-select active checkmark and its row label; the checked multi-select checkbox; the "Xong (N)" commit button; the "Hôm nay" shortcut; the trigger's focus ring and the search field's focus border; the calendar's today/selected cell. Not applied to every row, every border, or idle text.

Supporting neutrals (borrowed from existing `Form/Select.tsx` so converted screens match):
- Trigger border idle `#d9d9d9` (AntD default input border), hover `#7436dc`, focus ring `0 0 0 2px rgba(116,54,220,0.12)`.
- Error state (status==='error'): border `#ff4d4f`, focus ring `0 0 0 2px rgba(255,77,79,0.12)` — mirrors the AntD error ring so `Form` validation reads identically (D-02).
- Placeholder / muted text `#bfbfbf` (AntD default placeholder). Secondary caption text `#64748b`.
- Disabled trigger: background `#f5f5f5`, text `rgba(0,0,0,0.25)`, border `#d9d9d9` (AntD disabled input look, D-03).
- Row divider (between option rows / action rows): `1px solid #f0f0f0`.

## Copywriting Contract

Vietnamese, friendly-familiar tone (matches the v1.0 voice pass: `Hủy` not `Huỷ`, warm not admin). All literals belong in `AppCopy` when wired (Phase 10), but Phase 8 defines the canonical strings the components ship with.

| Element | Copy |
|---------|------|
| Primary CTA (multi-select / range commit) | `Xong (N)` where N = count of items in the draft (e.g. `Xong (3)`); plain `Xong` when N = 0 or for range when both ends chosen |
| Secondary / cancel button (multi-select, range, ActionMenu) | `Hủy` |
| Single-select clear affordance (`allowClear`) | `Bỏ chọn` (a leading row that sets value to `undefined` and dismisses) |
| Date quick-action | `Hôm nay` (sets today, clamped to min/max) |
| Multi-select selected-summary header (in-sheet) | `Đã chọn (N)` (reuses the existing `Form/Select.tsx` panel wording) |
| Closed trigger — multi-select summary | first option label `+N` when count > 1, or `N đã chọn` when labels don't fit (e.g. `Rau củ +2` or `3 đã chọn`) — matches D-04 |
| Search field placeholder | `Tìm kiếm` |
| Search empty result | Heading `Không tìm thấy` · Body `Thử từ khóa khác` |
| Option list empty (no options at all) | Heading `Chưa có lựa chọn` · Body `Danh sách đang trống` |
| Trigger placeholder (empty value) | Falls through to the call site's `placeholder` prop; component default when none passed: select `Chọn` · date `Chọn ngày` · range `Chọn khoảng ngày` |
| Error state | No new error copy — the picker renders AntD's existing error ring/aria from `status==='error'`; the validation message stays owned by the `Form.Item` rule (D-02). Trigger sets `aria-invalid` and the red ring; message text is the form rule's. |
| Destructive confirmation (ActionMenu) | ActionMenu does not self-confirm — a `danger` row runs its `onClick` (the call site owns any follow-up confirm Sheet). Destructive rows render label in `#ff4d4f`. `Hủy` is the detached escape, below the group. |

Empty-state note: SheetSelect/MultiSelect are "no data" surfaces, not full screens — empty copy renders centered in the sheet body at 14px `#64748b`, not as a hero. SheetActionMenu and SheetDatePicker have no empty state (actions/calendar are always present).

## Component Interaction Contracts

Prescriptive specs for the four pickers. These are the load-bearing visual/interaction contracts the planner and executor consume.

### Shared — Trigger (closed state, D-03/D-04)
- Dedicated button styled as an AntD input box: `min-height:44px`, `padding:0 11px`, `border:1px solid #d9d9d9`, `border-radius:6px`, `background:#fff`, full width, left-aligned text 16px/400.
- Shows: selected label (single) · `first +N` or `N đã chọn` (multi) · formatted date/range (date) · muted `#bfbfbf` placeholder when empty.
- Trailing affordance: clear `×` (`CloseCircleFilled`, `rgba(0,0,0,0.25)`, hover `rgba(0,0,0,0.45)`) when `allowClear` and a value exists; otherwise the type's idle glyph (`DownOutlined` for select, `CalendarOutlined` for date) in `#bfbfbf`. The clear × calls `onChange(undefined)` and does NOT open the sheet (stops propagation).
- Forwards injected `id`; sets `aria-haspopup="dialog"`, `aria-expanded`, and `aria-invalid` when `status==='error'`. Focus ring `0 0 0 2px rgba(116,54,220,0.12)` (or red ring on error). Disabled look per Color. Is the focus-return target the Sheet restores to on close.

### SheetSelect (PICK-01/02, D-05)
- Option rows: `role="option"`, `min-height:44px`, `padding:11px 16px`, label 16px/400 left, selected row shows a trailing `CheckOutlined` in `#7436dc` and label also `#7436dc`; `aria-selected` set (not checkmark-only — a11y, PITFALLS §E).
- Tap row → `onChange(value)` → sheet auto-dismisses. No commit button.
- `allowClear` → a leading `Bỏ chọn` row (same 44px metrics, label `#64748b`) → `onChange(undefined)` → dismiss.
- Search field appears when `options.length >= 8` OR `showSearch` passed (threshold locked at 8). Field is an AntD `<Input>` with `SearchOutlined` prefix, `Tìm kiếm` placeholder, 16px text, pinned at the TOP of the sheet body (PITFALLS §4 — keep above the iOS keyboard), `margin:8px 0 12px`. Filter is client-side, diacritic-insensitive (NFD strip + explicit đ/Đ).
- Pressed/hover row: background `rgba(116,54,220,0.06)`.

### SheetMultiSelect (PICK-03/04, D-06)
- Checkbox rows: AntD `<Checkbox>` (carries `aria-checked`) + label, `min-height:44px`, `padding:11px 16px`, checked fill `#7436dc`. Sheet STAYS OPEN; toggles mutate a local draft only.
- Sticky footer via Phase-7 `SheetActions`: primary `Xong (N)` (`#7436dc` fill, white 16px/600) commits `onChange(draft)` + dismiss; secondary `Hủy` reverts (discards draft, never calls `onChange`).
- While draft ≠ value-on-open (dirty): pass `maskClosable={false}` to `<Sheet>` so an accidental drag-dismiss springs back (Phase 7 D-04). Explicit `Hủy` still closes.
- Optional `Đã chọn (N)` summary strip at top (`#f8fafc`, 14px/600 `#64748b`) mirroring `Form/Select.tsx`.
- Same search rule/threshold as SheetSelect.

### SheetDatePicker (PICK-05/06, D-08/D-09)
- Hosts AntD's own `<DatePicker>`/`<RangePicker>` panel inside the sheet via `open` forced true + `getPopupContainer={() => sheetBodyRef}`; neutralize the global `z-index:4200` with `popupStyle={{ position:'static', zIndex:'auto', boxShadow:'none' }}` and a scoped `popupClassName` so the panel flows inline (Pitfall 3). `viVN` locale + `dayjs.locale('vi')` already global. Value stays `Dayjs`.
- `Hôm nay` is a prominent quick-action row/button (16px/600, accent `#7436dc`) above or beside the panel → `dayjs()` clamped to min/max (`disabledDate(today) ? nearest allowed : today`).
- Single date/datetime: commit on panel change → auto-dismiss (matches single-select feel); for `showTime`, let the confirm fire `onChange` then close (prefer the unified `SheetActions` "Xong" over AntD's inline OK footer for sheet consistency).
- Range mode: drives a draft like multi-select; commits only on `Xong` via `SheetActions` so a half-picked range can't escape; `Hủy`/dismiss reverts. `inputReadOnly` on the hidden input to suppress the iOS keyboard.
- `RangePicker` is a sub-export: `Object.assign(SheetDatePicker, { RangePicker })` (mirrors `Form/DatePicker.tsx`, D-01).

### SheetActionMenu (PICK-07, D-10)
- iOS grouped action sheet: action rows in ONE rounded surface (`background:#fff` on the `#f8fafc`-tinted body, `border-radius:14px`, rows divided by `1px #f0f0f0`). Each row full-width, `min-height:44px`, centered or leading-icon-aligned label 16px, optional leading icon (`md`/16px gap).
- `danger` row → label + icon `#ff4d4f` (weight stays 400). Each row's `onClick` runs then closes.
- `Hủy` is a SEPARATE, visually detached button BELOW the group (`lg`/24px gap), full-width, its own rounded surface (`#fff`, 14px radius), label 16px/600. Rendered via `SheetActions`.
- Not Form-bound (it's an action dispatcher, not a value field) — still hosted in `<Sheet>`.
