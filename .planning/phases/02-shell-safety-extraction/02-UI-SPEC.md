# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note (read first)

Phase 2 is **primarily a mechanical extraction refactor**, not visual design work. The bottom-tab navigator, cooking pill, sidebar drawer, global search, and data backup move out of `src/Routing/MasterPage.tsx` into `src/Routing/Shell/` with behavior **verified identical** (proven by Playwright e2e before/after, D-07). **Do not invent new visual design for moved components** — their pixels, copy, and `data-testid` hooks (D-08) must match `main` byte-for-byte.

This contract therefore covers only the **two genuinely net-new UI surfaces**:

1. **Error-boundary recovery UI** (FND-01, D-01/D-02) — the fallback shown instead of a white screen when a render error is thrown.
2. **`@components/Sheet`** (FND-02 / MOB-03 build-only, D-09/D-10) — a bottom-sheet wrapper over the existing `FastOverlay` portal system (NOT antd `Drawer` — conscious D-09 divergence), shipped as component + API only; first real consumers arrive in Phase 5.

All token tables below are the **existing app contract** (sourced from `src/App.tsx` `ConfigProvider` and `FastOverlay.tsx`), recorded so the two net-new surfaces match the shell they live in. They are not new decisions.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, Sheet body padding |
| lg | 24px | Section padding, error-fallback vertical rhythm |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions (carried from the existing `FastOverlay` shell so net-new surfaces match it — not new design):

- **12px / 14px** overlay header paddings — `FastDrawerShell` uses `padding: "14px 14px 12px 16px"` and `gap: 12`. `Sheet`'s header must mirror this exact rhythm to sit consistently with the drawer; do not "round" to 16px.
- **18px border-radius** — overlay panels use `18px`. `Sheet` uses `borderRadius: "18px 18px 0 0"` (rounded **top** corners only, since it pins to the bottom).
- **44px minimum touch target** — interactive controls in `Sheet` (close button, future picker rows) target ~44px to satisfy MOB-02 when Phase 5 consumers arrive. Applies to `Sheet` only; moved components keep their current sizing.

## Color

The app theme is a warm purple system (`colorPrimary: #7436dc`). The 60/30/10 split below describes the two net-new surfaces against the existing shell.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Panel/surface background (Sheet panel, error-fallback card body) |
| Secondary (30%) | `linear-gradient(180deg, #f5f0ff 0%, #ffffff 42%)` | Sheet panel gradient + soft header tints, matching `FastDrawerShell` |
| Accent (10%) | `#7436dc` (hover `#8f46f7`, active `#5e2bbf`) | See reserved list below |
| Destructive | `#ff4d4f` (antd `colorError` default) | Not used by net-new Phase 2 surfaces — see note |

Accent reserved for: the **Reload button** in the error fallback (antd primary `Button`, inherits `colorPrimary`), and **focus/active affordances on the `Sheet` close button**. Not applied to body text, surfaces, or non-interactive chrome.

Overlay backdrop: `rgba(16, 24, 40, 0.30)` — `Sheet` reuses the `FastOverlay` backdrop tone (the drawer uses `0.30`; a sheet may use up to `0.36` per the research sketch) so stacking reads consistently.

Destructive note: Phase 2 introduces **no net-new destructive actions**. The only destructive-capable surface is the `DataBackup` import (overwrites `persist:personal`), which moves **intact** (D-06) — its existing confirmation is preserved verbatim, not redesigned here.

## Copywriting Contract

Phase 2 owns exactly one net-new piece of user-facing copy: the **error-boundary fallback**. Copy must match the app's warm Vietnamese register ("nhà mình" / "nhé"). The error boundary must **not** render `error.message` or stack traces to users (V7 security: keep the message generic). Seed via `AppCopy` if a natural namespace fits; otherwise inline and defer formal migration to Phase 5 (COPY-03).

| Element | Copy |
|---------|------|
| Primary CTA | **Tải lại trang** (reload — antd primary `Button` with `ReloadOutlined`, `onClick={() => window.location.reload()}`, D-02) |
| Empty state heading | Not applicable — Phase 2 ships no data-bearing screens (`Sheet` is an empty wrapper; extraction shows no new empty states) |
| Empty state body | Not applicable this phase (friendly empty-states are Phase 5 / COPY-05) |
| Error state | Heading: **"Ứng dụng gặp chút trục trặc rồi"** · Body: **"Nhà mình thử tải lại trang nhé, dữ liệu vẫn được giữ an toàn."** + Reload CTA. Generic, no technical detail / stack trace surfaced. |
| Destructive confirmation | None net-new this phase. `DataBackup` import confirmation moves intact (D-06) — copy unchanged. |

Existing `AppCopy.common.error` (`"Lỗi hệ thống, vui lòng thử lại sau"`) is more technical/clinical than the warm register above; the fallback copy here is intentionally friendlier. The executor may add a `AppCopy.common.crashTitle` / `crashBody` / `reload` triple if seeding fits cleanly, or inline the strings and let Phase 5 reconcile the namespace.

## Net-New Surface Specs

### 1. Error-boundary recovery UI (FND-01)

- **Mount:** one top-level boundary around `<RootRouter />`, **inside `ConfigProvider`** (D-01) so the fallback inherits the antd theme (`#7436dc`) and `viVN` locale. Mounting it above `ConfigProvider` would strip styling/locale (research Pitfall 4).
- **Layout:** centered card on a white/`#f5f0ff` surface. Vertical stack: display heading (28px/600) → body line (18px/400, line-height 1.5) → primary Reload button. Use `lg` (24px) vertical rhythm, `md` (16px) inner padding. Comfortable on both phone and desktop (single column, max content width ~480px, centered).
- **Recovery affordance:** single antd primary `Button` (`ReloadOutlined` + "Tải lại trang") calling `window.location.reload()` (D-02). No granular in-place reset.
- **Catch scope:** render/lifecycle errors only (class component, `getDerivedStateFromError`). Frame success as "a *thrown render error* shows recovery UI" (research Pitfall 5). Do not surface `error.message` to the user.

### 2. `@components/Sheet` (FND-02 / MOB-03 build-only)

- **Implementation:** bottom-placement variant on the existing `FastOverlay` system (D-09) — NOT antd `Drawer`. Reuse `useBodyScrollLock`, `useEscapeClose`, `useResolvedOverlayZIndex`, and the shared `allocateOverlayStackToken` singletons so z-index stacking, scroll-lock, escape-to-close, and `prefers-reduced-motion` behavior match `FastDrawerShell`/`FastModalShell`. Co-locate in / re-export from `FastOverlay.tsx` to avoid forking the stacking-token singletons (research Pitfall 3).
- **Placement:** panel pinned to bottom, full-width up to `maxWidth: 720px`, centered; `borderRadius: "18px 18px 0 0"`; `maxHeight: "min(85vh, 720px)"`; slide-up (`translateY`) enter animation that respects reduced-motion.
- **Surface:** white panel body with the `#f5f0ff → #ffffff` gradient header tint matching the drawer; header row = title (20px/600, mirrors `shellTitleStyle`) + close button (`CloseOutlined`, ~44px target, `aria-label`).
- **A11y:** `role="dialog"`, `aria-modal="true"`, `maskClosable` + `keyboard` (escape) defaults to `true`, accepts a `data-testid` passthrough (consistent with `FastDrawerShell`).
- **API surface (Claude's discretion, D-09):** at minimum `{ open, title, onClose, children, height?, zIndex?, maskClosable?, closable?, keyboard?, "data-testid"? }` — designed to cover Phase 5's picker + confirmation needs. Barrel export `@components/Sheet`.
- **Scope guard (D-10):** wrapper + minimal mount/open smoke proof only. **Do not** migrate any existing picker/confirmation onto it this phase.

### Extracted (moved) components — identity constraint

`BottomTabNavigator`, `CookingPill`, `SidebarDrawer` (incl. backup-center modal, admin, PIN), `PageActionsMenu`, `DataBackup`, global search overlay: **visual output and all `data-testid` hooks must be identical to `main`** (D-05, D-08). No restyling, relabeling, copy migration, or spacing/color "cleanups" during the move. The reviewer's diff should read as a move, not a rewrite. Identity is gated by Playwright e2e before/after (D-07) — and the baseline must be confirmed green first (research Pitfall 1).
